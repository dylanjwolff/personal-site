About me

I'm a graduate student at ETH Zurich pursuing a masters in Computer Science with a focus in Information Security. Specifically I am interested in leveraging static and dynamic program analysis (as well as verification) techniques to find and prevent bugs in software. Before starting my masters, I worked at Mathworks on a variety of projects, mostly involving testing and deployment infrastructure for web applications.

About this site

To build this site, I used a simple, free graphical tool called Mobirise. I think that there is often a stigma among programmers about using these kinds of tools, but I do believe that for most (non-frontend) software engineers, it absolutely makes sense to design interfaces in a graphical environment. Doing so typically results in a more coherent design, and frees up time to focus on projects that you care more about.

This site's hosting, I think, is much more interesting. I am currently self hosting on a bare-metal Kubernetes cluster within my home network. The cluster is only two ARM-64 Raspberry Pi nodes right now, but I'm hoping to be able to expand the cluster soon! While this setup is clearly not practical, it's a great opportunity for experimentation, and it allows me to keep up with the Kubernetes ecosystem.

Other Projects

Rust PCS

Research Project

Formalized the notion of a "Place Capabilities Summary" to represent memory access permissions for Rust programs. Leveraged information from the Polonius borrow-checker and the Rust compiler itself to create a PCS API for use by verification software and other Program Analysis tools.

Mutational Fuzzer

Bachelor's Thesis

Developed a distributed, mutational fuzzer in Python for Windows applications using the WinAppDbg debugging library. Found 10 unique bugs (by instruction pointer) in VLC media player. Assesed the impact of tuning various parameters on fault detection rate.
